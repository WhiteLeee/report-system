# 慧运营整改单自动下发需求文档

## 1. 背景

当前 report-system 的整改单下发依赖人工在巡检结果处理页中逐条确认后提交。随着报告量增加，人工确认再下发的方式会带来明显的操作成本和响应延迟。

为提升闭环效率，需要在 report-system 中新增“整改单自动下发”能力，在报告发布成功后，系统根据规则自动判断哪些巡检结果需要创建整改单，并调用慧运营 API 完成下发。

## 2. 目标

本次实现目标：

1. 支持在系统设置中开启或关闭整改单自动下发。
2. 支持在报告发布成功后自动执行整改单下发。
3. 自动下发流程与人工复核提交流程解耦。
4. 为后续增加更多自动下发规则保留扩展点。

## 3. 范围

本次范围内：

1. 发布报告后按规则自动扫描巡检结果。
2. 命中规则的结果自动创建慧运营整改单。
3. 支持在系统设置中配置自动下发开关、触发方式、规则、执行人和重复建单策略。
4. 发布接口返回自动下发执行摘要。

本次范围外：

1. 自动下发失败后的后台重试队列。
2. 自动下发专用批次日志页面。
3. 按组织、企业、门店单独配置不同规则。
4. 自动下发成功后自动修改结果复核状态。

## 4. 核心业务原则

1. 自动下发不影响报告成功入库。
2. 自动下发失败不应导致 `/api/reports/publish` 返回失败。
3. 自动下发与人工复核是两条独立链路。
4. 自动下发只负责创建整改单，不默认变更结果复核状态。

## 5. 配置项

在“系统设置 -> 整改单设置”中新增：

1. 自动下发整改单
   - `关闭`
   - `发布后自动下发`

2. 自动下发规则
   - `有问题项时自动下发`

3. 自动下发执行人
   - 默认：`auto-rectification`

4. 已有整改单时
   - `跳过当前结果`
   - `允许重复创建`

5. 自动备注
   - 可选文本
   - 自动拼接到整改单描述中

## 6. 默认规则

当前默认规则为：

1. 对报告中的每条巡检结果进行扫描。
2. 当结果命中 `issue_found` 语义，且存在问题项时，自动创建整改单。
3. 整改单图片使用当前巡检结果图片。
4. 整改单描述由“问题项列表 + 自动备注”拼接生成。
5. 整改截止日期使用系统设置中的默认整改天数。
6. 当同一结果已存在整改单，且配置为跳过时，不重复创建。

## 7. 触发时机

当前支持：

1. `发布后自动下发`
   - 报告发布成功并入库后立即执行。

后续可扩展：

1. 自动复核完成后自动下发
2. 定时补扫未建单结果
3. 人工复核提交时自动跳过确认弹窗

## 8. 执行结果

每次报告发布后，发布接口返回自动下发摘要：

1. 是否启用
2. 触发方式
3. 扫描结果数
4. 创建成功的结果数
5. 创建成功的整改单数
6. 跳过数
7. 失败数

## 9. 迭代方向

后续规则扩展建议包括：

1. 按问题严重级别自动下发
2. 按组织或门店类型选择性下发
3. 按 inspection 规则名选择性下发
4. 失败任务重试
5. 自动下发批次日志与审计页面
